@keyframes (Visualizador)

Exemplos práticos de animações com @keyframes + animation: mover, fade, escala, rotação, cor, composição, loading, pulso, slide, steps e bounce.

Dica: no seu sistema, ative “Reduzir movimento” pra ver o prefers-reduced-motion funcionando.

1) Movimento (translate)

Use transform: translate() para melhor performance.

Mover na horizontal

MOVE

animation: moveX 1.8s ease-in-out infinite alternate;

“Bounce” simples

animation: bounce 1.2s ease-in-out infinite;

2) Opacidade (fade)

Fade-in/out

FADE

animation: fade 1.4s ease-in-out infinite;

Pulso (scale + opacity)

animation: pulse 1.5s ease-in-out infinite;

3) Escala (scale)

Scale loop

SCALE

animation: scaleLoop 1.6s ease-in-out infinite;

Entrada (slide + fade)

Entrando...

animation: slideIn 0.7s ease-out both;

4) Rotação (rotate)

Rotação contínua

ROTATE

animation: spin 1.4s linear infinite;

Spinner (loading)

animation: spin 0.9s linear infinite;

5) Cor (background-color)

Troca de cor

COLOR

animation: colorShift 2.6s linear infinite;

Gradiente animado (background-position)

animation: gradientMove 2.5s ease-in-out infinite;

6) Animação composta

Combine transform + opacity para animações suaves.

Transform + opacity

COMBO

animation: combo 3s ease-in-out infinite;

Saída (slide-out)

Saindo...

animation: slideOut 0.7s ease-in both;

7) steps() (animação “em quadros”)

Indicador “digitando”

steps() é ótimo para efeitos tipo “máquina de escrever”.

...

animation: dots 1.2s steps(4, end) infinite;

8) Observações de performance e acessibilidade

  • Prefira animar transform e opacity.
  • Evite animar frequentemente width, height, top/left.
  • Respeite prefers-reduced-motion para usuários sensíveis a animações.
